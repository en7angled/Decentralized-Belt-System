# Add Documentation

## Overview

Add documentation to Haskell code in the Decentralized Belt System using Haddock conventions and project standards.

## Steps

1. **Module-Level Documentation**
    - Every module must have a Haddock module header: `-- | Module description`
    - Describe the module's purpose and its role in the architecture (which layer, what it provides)
    - Note key dependencies and which other modules consume this one

2. **Export Documentation (Haddock `-- |`)**
    - Add `-- |` comments above exported functions and types
    - Document purpose, parameters, return value, and any preconditions
    - For smart constructors (`mk` prefix), document what invariants they enforce
    - For conversion functions, document source and target layers

3. **Record Field Documentation (Haddock `-- ^`)**
    - Add `-- ^` comments on record fields where the meaning isn't obvious from the name
    - Always document fields with non-obvious units, formats, or constraints
    - Example: `-- ^ IPFS CID for the profile image (without gateway prefix)`

4. **Prose Documentation (docs/)**
    - For complex workflows, write narrative docs in `docs/` (e.g., CR-*.md change requests)
    - Document the "why" — design decisions, trade-offs, constraints
    - Include sequence diagrams (Mermaid) for multi-step flows
    - Keep `docs/onchain-trace-codes.md` up to date when adding new trace codes

5. **What NOT to Document**
    - Do not write Swagger docs by hand — they are auto-generated on API startup to `docs/generated/swagger/`
    - Do not write Blueprint docs by hand — generated by `admin write-blueprint`
    - Do not add comments that restate the code (`-- increment x by 1`)
    - Do not add documentation to code you didn't change

## Add Documentation Checklist

- [ ] Module-level Haddock header present
- [ ] Exported functions and types have `-- |` documentation
- [ ] Non-obvious record fields have `-- ^` annotations
- [ ] Complex workflows documented in `docs/` with diagrams
- [ ] `docs/onchain-trace-codes.md` updated if new trace codes added
- [ ] No hand-written Swagger or Blueprint docs
- [ ] `cabal build all` succeeds (Haddock syntax is valid)
